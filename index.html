Proyectos

Proyecto #1 - Urban Scooter

Web Testing, Mobile testing, API testing

Este proyecto de QA para el sitio web de Urban Scooter involucró: Elaboración de una lista de comprobación de los requisitos de función de la pantalla "Estado del pedido", validación a los campos para la pantalla "Hacer pedido", validación de diseños contra especificaciones de Figma, creación de casos de prueba, ejecución de pruebas manuales, revisión de los requisitos del back-end y la documentación de la API, Diseño de lista de comprobación y prueba la API frente a los requisitos, asegurando una comunicación correcta entre los servicios. Identifiqué y reporté más de 80 bugs en Jira.

JIRA, Postman, JSON, ApiDoc

Proyecto #2 - Urban Routes

Web Testing

Este proyecto de QA para el sitio web de Urban Routes involucró: Validación de diseños contra especificaciones de Figma, validación de las ventanas "Método de pago" y "Agregar tarjeta", creación de casos de prueba manuales dela lógica del botón "Reservar", creación de casos de pruebas manuales la lógica de las funciones de reserva. Identifiqué y reporté más de 60 bugs en Jira.

JIRA

Proyecto #3 - Urban Grocers

API Testing

Analicé los requisitos para la nueva funcionalidad del back-end de Urban Grocers para diseñar una lista de comprobación orientada a realizar pruebas para las funcionalidades de "Agregar comestibles a un kit" y comprobar si el servicio de entrega se encuentra disponible y cual es su costo; del mismo modo diseñé y ejecuté casos de prueba a su API a través de Postman, realizando los reportes en JIRA de más de 20 bug hallados.

Postman, JIRA, JSON, ApiDoc
